How It Works

Students aren't just learning code — they're making creative decisions every step of the way.

Step 01
🎮

Explore Classic Games

Day 1 begins with a guided tour of iconic 2D games. Students play and analyze what makes each one fun and how it might be built.

Step 02
🗳️

Class Vote

Students vote on which game the class will build together. The winning game sets the direction for all four weeks.

Step 03
💻

Code Week by Week

Starting from a blank file, the game grows each week — a working prototype, then features, then polish — using JavaScript and p5.js.

Step 04
🏆

Final Game Showcase

Week 4 ends with a live demo event. Students present their game to peers, add personal touches, and celebrate what they built.

4-Week Plan

Week-by-Week Curriculum

Each week builds on the last, moving from concept to code to a fully playable, polished game.

Week 1 · Days 1–5
Explore, Vote & Set Up
Students play classic games, choose their favorite, and get their dev environment ready.
🎮
Monday · Day 1
Game Exploration
  • Play 8 demo games live in browser
  • Discuss what makes each game fun
  • Identify patterns: loops, collision, score
  • Class vote — announce the winning game!
Tuesday · Day 2
Dev Environment Setup
  • Create a p5.js Web Editor account
  • JavaScript basics refresher
  • Tour the editor: code, preview, console
  • "Hello, canvas!" — first shapes on screen
Wednesday · Day 3
The Game Loop
  • What is a game loop? (setup → draw)
  • Frames per second & timing
  • Draw a shape, move it with arrow keys
  • Mini-challenge: change color on key press
Thursday · Day 4
Deconstruct Our Game
  • Sketch the game on the whiteboard
  • List every object: player, walls, enemies
  • Identify each system needed
  • Create a project file structure
Friday · Day 5
Player Object
  • Define the player as a JavaScript object
  • Add position, speed properties
  • Keyboard input → movement
  • Keep player inside the canvas (bounds)
Week 1 Outcomes
Game Loop JavaScript Objects Keyboard Input p5.js Setup Web Editor Project Decomposition
Week 2 · Days 6–10
Core Mechanics
Build the game's engine: movement, collision detection, and a working prototype.
⚙️
Monday · Day 6
Sprites & Images
  • Load images with loadImage() and preload()
  • Draw sprites with image()
  • Replace placeholder shapes with art
  • Discuss copyright-free game assets
Tuesday · Day 7
Other Game Objects
  • Create enemy / ball / block objects
  • Spawn multiple objects with an array
  • Animate using frameCount & timers
  • Debug with console.log()
Wednesday · Day 8
Collision Detection
  • Bounding-box vs. circle collision
  • Using dist() and overlap checks in p5.js
  • What happens on hit? (die, bounce, score)
  • Test + fix edge cases
Thursday · Day 9
Scoring & State
  • Display score with font rendering
  • Game states: PLAYING / GAME OVER
  • Conditionals to switch states
  • "Play Again" key press logic
Friday · Day 10
Prototype Demo Day
  • Finish playable prototype (any state)
  • Share screen — classmates play it!
  • Give + receive feedback cards
  • Plan Week 3 feature additions
Week 2 Outcomes
Collision Detection Sprites & Assets Arrays & Loops Game States Score Display Working Prototype
Week 3 · Days 11–15
Features & Game Logic
Add what makes the game exciting — levels, audio, difficulty scaling, and extra mechanics.
🚀
Monday · Day 11
Sound & Music
  • p5.sound library & loadSound()
  • Load .mp3 / .wav audio files
  • Trigger sounds on events (hit, score, die)
  • Background music loop
Tuesday · Day 12
Difficulty Scaling
  • Increase speed over time
  • Spawn more enemies by score tier
  • Variables vs. hardcoded values
  • Balancing: playtesting rounds
Wednesday · Day 13
Levels or Waves
  • Design 3 level/wave patterns
  • Level transition screen
  • Array/object-driven level data
  • Win condition when all levels done
Thursday · Day 14
Player Choice Day
  • Students add one personal feature
  • Power-ups, skins, extra lives, etc.
  • Pair-programming encouraged
  • Code review with teacher
Friday · Day 15
Beta Test Day
  • Swap games with a classmate
  • Find and report 3 bugs each
  • Fix highest-priority bugs
  • Git commit: "beta build"
Week 3 Outcomes
Sound Effects Difficulty Design Level Systems Objects Bug Testing Git Commits
Week 4 · Days 16–20
Polish & Launch
The final sprint — screens, effects, README, and a live showcase for friends and family.
🏆
Monday · Day 16
Title & Menu Screens
  • Design a title screen layout
  • Animated title text with text() and sin()
  • Menu: Start, How to Play, Quit
  • Consistent visual theme/color palette
Tuesday · Day 17
Visual Polish
  • Particle effects (explosions, sparks)
  • Screen shake on impact
  • Animated sprites (walk cycle)
  • Background parallax layer
Wednesday · Day 18
High Scores & Save
  • Save scores with localStorage
  • Read + display top 5 scores
  • Name entry screen (initials)
  • Discuss browser storage basics
Thursday · Day 19
Share & Publish
  • Generate a shareable p5.js sketch link
  • Push code to a public GitHub repo
  • Write a project README.md
  • Peer-play all finished games
Friday · Day 20
🎉 Showcase Day!
  • Live demo to audience
  • Each student explains one piece of code
  • Best Feature + Most Fun awards
  • Reflection: what would you add?
Week 4 Outcomes
Screen Design Particle Effects Browser Storage README Writing Public GitHub Repo Live Demo Skills
Tech Stack

Tools Students Use

The same tools used by real web developers — inside a beginner-friendly, browser-based workflow.

✨
JavaScript
Primary coding language — the language of the web, runs in every browser
🎮
p5.js
Creative coding library — drawing, animation, sound, and input made simple
🖥️
p5.js Web Editor
Browser-based coding environment — write code, hit play, share with one link
🌐
Web Browser
The game runs right in the browser — no installs, no downloads, instant results
🐙
GitHub
Version control + project hosting — students save work and share their code
🎨
Piskel / LibreSprite
Pixel art editor — students create or edit their own sprites and tilesets
